# OpenCode Native Workflow Pack

AgentFlow cannot currently programmatically create a clickable OpenCode native subagent task. OpenCode native UI tasks must be created by OpenCode itself. The workflow pack is the honest bridge: AgentFlow prepares the work package, and OpenCode native subagents execute it.

## What AgentFlow Does

`workflow:native-pack` creates:

```text
.workflow-runs/<runId>/native-subagents/
  DISPATCH.md
  manifest.json
  planner/
    input.json
    prompt.md
    output.schema.json
  debater/
    input.json
    prompt.md
    output.schema.json
  planner-revision/
    input.json
    prompt.md
    output.schema.json
  executor/
    input.json
    prompt.md
    output.schema.json
  verifier/
    input.json
    prompt.md
    output.schema.json
```

Each native task records role, OpenCode agent name, input path, output path, expected schema, success criteria, and dependencies.

## What OpenCode Does

OpenCode should create real native subagent tasks, for example:

- `@agentflow-planner`
- `@agentflow-debater`
- `@agentflow-planner-revision`
- `@agentflow-executor`
- `@agentflow-verifier`

Each subagent reads its `input.json`, follows its `prompt.md`, and writes exactly one `output.json` at the path assigned in the pack. If OpenCode creates the subagent task, the UI can show that native subagent's work process.

## Pack Command

In OpenCode, use:

```text
/agentflow native-pack 解释一下咖啡的做法
```

This calls the MCP tool `agentflow_native_pack` and returns a copyable dispatch prompt for OpenCode native subagents.

CLI fallback:

```bash
npm run workflow:native-pack -- --profile agent-workforce-basic --task "解释一下咖啡的做法"
```

The command prints `runId`, `dispatchInstructionsPath`, `manifestPath`, task count, and each native task.

## Dispatch Instructions

`DISPATCH.md` tells the user:

1. Which OpenCode native subagents to create.
2. Which `@agentflow-*` agent to use for each role.
3. Which `input.json` each subagent reads.
4. Which `output.json` each subagent writes.
5. The dependency order.
6. The collection command to run after outputs exist.

## Collect Command

In OpenCode, use:

```text
/agentflow native-collect <runId>
```

This calls the MCP tool `agentflow_native_collect`.

CLI fallback:

```bash
npm run workflow:native-collect -- --run <runId>
```

The collector reads each assigned `output.json`, validates it with `SchemaValidator`, writes `trace.json`, and writes a summary.

If an `output.json` is missing, the corresponding role stays `pending`:

```text
Planner
  source: opencode_native_artifact
  status: pending
  reason: output artifact not found
```

The collector never fabricates missing agent output. A role is completed only when its output artifact exists and validates.

## Difference From AgentFlow Internal Subagents

AgentFlow internal subagents are runtime artifacts generated by `SubAgentDispatcher`.

OpenCode native subagents are OpenCode-created tasks visible in the OpenCode UI. The workflow pack does not claim OpenCode created those tasks; it only provides a structured dispatch package so OpenCode can create them natively.
